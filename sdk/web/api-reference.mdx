This page provides detailed documentation for all Linkrunner Web SDK methods, interfaces, and configuration options.

## SDK Instance

The SDK is available globally as `window.LinkrunnerSDK` when loaded via CDN, or can be imported when using package managers.

### Global Access

```javascript
// Available globally after CDN load
window.LinkrunnerSDK.init({ ... });

// Check if SDK is loaded
if (typeof window.LinkrunnerSDK !== 'undefined') {
  // SDK is available
}
```

### Module Import

```javascript
// ES Module import
import { init, capturePageView, getState } from '@linkrunner/web-sdk';

// CommonJS import
const { init, capturePageView, getState } = require('@linkrunner/web-sdk');
```

## Core Methods

### init()

Initialize the Linkrunner SDK with your configuration.

<ResponseField name="init" type="function">
  ```typescript
  init(options: InitOptions): Promise<InitResponse>
  ```
  
  **Parameters:**
  
  <ParamField body="options" type="InitOptions" required>
    Configuration object for SDK initialization
    
    <Expandable title="InitOptions Properties">
      <ParamField body="token" type="string" required>
        Your Linkrunner API token
      </ParamField>
      
      <ParamField body="apiUrl" type="string">
        Custom API endpoint URL. Defaults to `https://api.linkrunner.io`
      </ParamField>
      
      <ParamField body="debug" type="boolean">
        Enable debug logging. Defaults to `false`
      </ParamField>
      
      <ParamField body="appVersion" type="string">
        Your application version. Defaults to `'1.0.0'`
      </ParamField>
      
      <ParamField body="platform" type="string">
        Platform identifier. Defaults to `'web'`
      </ParamField>
      
      <ParamField body="source" type="string">
        Traffic source identifier (e.g., 'google-ads', 'organic')
      </ParamField>
      
      <ParamField body="link" type="string">
        Campaign or link identifier
      </ParamField>
      
      <ParamField body="timeout" type="number">
        Request timeout in milliseconds. Defaults to `10000`
      </ParamField>
    </Expandable>
  </ParamField>
  
  **Returns:**
  
  <ParamField body="Promise<InitResponse>" type="Promise">
    <Expandable title="InitResponse Properties">
      <ParamField body="session_id" type="string">
        Unique session identifier
      </ParamField>
      
      <ParamField body="config" type="SDKConfig">
        SDK configuration object
      </ParamField>
      
      <ParamField body="features" type="object">
        Available features for this session
      </ParamField>
      
      <ParamField body="endpoints" type="object">
        API endpoints configuration
      </ParamField>
    </Expandable>
  </ParamField>
</ResponseField>

**Example:**

```javascript
try {
  const response = await window.LinkrunnerSDK.init({
    token: 'your-api-token',
    debug: true,
    appVersion: '2.1.0',
    platform: 'web-ecommerce'
  });
  
  console.log('Session ID:', response.session_id);
  console.log('SDK Config:', response.config);
} catch (error) {
  console.error('Initialization failed:', error);
}
```

### capturePageView()

Track a page view event.

<ResponseField name="capturePageView" type="function">
  ```typescript
  capturePageView(url?: string): Promise<PageViewResponse>
  ```
  
  **Parameters:**
  
  <ParamField body="url" type="string">
    Optional URL to track. If not provided, uses `window.location.href`
  </ParamField>
  
  **Returns:**
  
  <ParamField body="Promise<PageViewResponse>" type="Promise">
    <Expandable title="PageViewResponse Properties">
      <ParamField body="success" type="boolean">
        Whether the page view was tracked successfully
      </ParamField>
      
      <ParamField body="message" type="string">
        Response message or error description
      </ParamField>
    </Expandable>
  </ParamField>
</ResponseField>

**Examples:**

```javascript
// Track current page
await window.LinkrunnerSDK.capturePageView();

// Track specific URL
await window.LinkrunnerSDK.capturePageView('/products/item-123');

// Track with full URL
await window.LinkrunnerSDK.capturePageView('https://example.com/checkout');

// With error handling
try {
  const response = await window.LinkrunnerSDK.capturePageView('/dashboard');
  if (response.success) {
    console.log('Page view tracked successfully');
  }
} catch (error) {
  console.error('Page view tracking failed:', error);
}
```

Get detailed health status of the SDK.

<ResponseField name="getHealthStatus" type="function">
  ```typescript
  getHealthStatus(): HealthStatus
  ```
  
  **Returns:**
  
  <ParamField body="HealthStatus" type="object">
    <Expandable title="HealthStatus Properties">

      </ParamField>

```javascript
  console.error('Last SDK Error:', health.lastError);
}
```

## Lifecycle Management

### reset()

Reset the SDK to uninitialized state while preserving install instance ID.

<ResponseField name="reset" type="function">
  ```typescript
  reset(): void
  ```
  
  **Description:**
  
  Clears session data and returns SDK to uninitialized state. The install instance ID is preserved for continuity.
</ResponseField>

### destroy()

Completely destroy the SDK instance and clear all storage.

<ResponseField name="destroy" type="function">
  ```typescript
  destroy(): void
  ```
  
  **Description:**
  
  Completely destroys the SDK instance, clears all storage including install instance ID, and removes event listeners.
</ResponseField>

**Example:**

```javascript
// Reset SDK (keeps install ID)
window.LinkrunnerSDK.reset();

// Reinitialize
await window.LinkrunnerSDK.init({ token: 'your-token' });

// Or completely destroy
window.LinkrunnerSDK.destroy();
```

## Auto-initialization Attributes

When using CDN with auto-initialization, these HTML attributes are supported:

| Attribute | Type | Description |
|-----------|------|-------------|
| `token` | string | **Required.** Your API token |
| `api-url` | string | Custom API endpoint URL |
| `debug` | boolean | Enable debug mode (`"true"` or `"false"`) |
| `platform` | string | Platform identifier |
| `app-version` | string | Application version |
| `package-version` | string | Override SDK version |
| `source` | string | Traffic source |
| `link` | string | Campaign identifier |
| `timeout` | number | Request timeout in milliseconds |

**Example:**

```html
<script 
  src="https://sdk.linkrunner.io/latest/linkrunner-sdk.min.js"
  token="your-token-here"
  api-url="https://api.linkrunner.io"
  debug="true"
  platform="web-ecommerce"
  app-version="2.1.0"
  source="google-ads"
  link="summer-campaign"
  timeout="15000"
></script>
```

## Error Handling Patterns

### Basic Error Handling

```typescript
try {
  await window.LinkrunnerSDK.init({ token: 'your-token' });
} catch (error) {
  if (error instanceof Error) {
    console.error('Init failed:', error.message);
  }
}
```

### Comprehensive Error Handling

```typescript
async function safeSDKOperation() {
  try {
    // Check SDK availability
    if (typeof window.LinkrunnerSDK === 'undefined') {
      throw new Error('SDK not loaded');
    }
    
    // Check SDK state
    if (!window.LinkrunnerSDK.isReady()) {
      throw new Error('SDK not ready');
    }
    
    // Perform operation
    const result = await window.LinkrunnerSDK.capturePageView();
    return result;
    
  } catch (error) {
    // Log error details
    const sdkError = window.LinkrunnerSDK.getLastError();
    if (sdkError) {
      console.error('SDK Error:', sdkError);
    }
    
    // Handle specific error types
    if (error instanceof Error) {
      switch (error.name) {
        case 'NetworkError':
          console.log('Network issue - will retry later');
          break;
        case 'TimeoutError':
          console.log('Request timed out - check connection');
          break;
        default:
          console.error('Unexpected error:', error.message);
      }
    }
    
    throw error;
  }
}
```

## Browser Compatibility

The SDK is built to work across all modern browsers. For specific compatibility requirements:

- **ES6+ Features**: Arrow functions, Promises, async/await, Classes
- **Web APIs**: Fetch API, localStorage, sessionStorage, URL constructor
- **Browser APIs**: navigator.userAgent, window.screen, Intl.DateTimeFormat

For older browser support, ensure polyfills are available for these features.
